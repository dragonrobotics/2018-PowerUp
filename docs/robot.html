robot module¶

The entry point for all robot code.

class robot.Robot[source]¶

Bases: wpilib.iterativerobot.IterativeRobot

autonomousInit()[source]¶

Called once when entering Autonomous mode.

autonomousPeriodic()[source]¶

Called periodically when in Autonomous mode.

To be specific, this function is called whenever a new packet has been received from the Driver Station.

Warning

This function must not block the robot program for more than a few milliseconds at a time. Blocking will cause all sorts of problems with the robot and may leave it unresponsive.

disabledInit()[source]¶

Called once whenever the robot is disabled.

This can’t, and probably shouldn’t, do much. It could be helpful to reload robot configuration values here (from Preferences, for instance) for testing purposes.

disabledPeriodic()[source]¶

Called periodically whenever the robot is disabled.

To be specific, this function is called whenever a new packet has been received from the Driver Station.

This probably shouldn’t do much other than update SmartDashboard.

robotInit()[source]¶

Perform initialization on robot startup.

This function is called when the robot code starts after being powered on; at this point everything within WPILib / NetworkTables / etc. has been initialized.
